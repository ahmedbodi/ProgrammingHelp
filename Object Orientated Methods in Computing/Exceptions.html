Exceptions:
    - Indicate problems that occur during a program’s execution
    - Occur infrequently

Exception Handling:
    - Can resolve exceptions
        - Allow a program to continue executing
        - Notify the user of the problem
        - Terminate the program in a controlled manner
    - Makes the program robust and fault-tolerant
    - A standard mechanism for processing errors
        - Especially important when working on a project with a large team of programmers
    - C++ has a mechanism for sending an exception signal up the call stack (things that need to be executed) regardless of intervening calls (things that are in line to be done first)
    - Use keyword throw followed by an operand
        - The throw operand can be of any type
        - If the throw operand is an object it is called an exception object

Traditional Exception handling:
    - Intermixing program and error handling logic
    - Makes the program difficult to read, modify, maintain and debug
    - Impacts performance

Stack unwinding:
    - Occurs if no matching catch handler is found
    - Program attempts to locate another enclosing try block in the calling function

Try Blocks:
    - Keyword “try”, followed by curly braces/brackets
    - Should enclose
        - statements that might cause exceptions
        - Statements that should be skipped in case of exception
       
Exceptions may surface:
    - Through explicitly mentioned code in a try block
    - Through calls to other functions and 
    - Through deeply nested function called initiated by code in a “try” block

Common Programming Errors:
    - Syntax error to place code between a try block and its corresponding catch handlers
    - Each catch handler can only have a single parameter
        - Specifying a comma operated list of exception parameters is a syntax error
    - Logic error to catch the same type in two different catch handlers follow a single try block

The C++ Standard library uses generic programming. 
    - Represents efficient algorithms in an abstract way    
        - Allowing it to apply to a wide range of data abstractions
    - Find general constraints
        - Allow the algorithms to perform efficiently
        - Maximise flexibility
    - Generic Programmers belief/experience
        - Many algorithms have the same constraints
        - Many data types satisfy these constraints

Finding Abstractions: Lifting
Lifting is a generalising concrete algorithms
    - Use template parameters to replace data types
    - Identify constraints on replacement types
    
How to do lifting:
    - Examine several versions of an algorithm
        - Look for common features
    - Produce generic algorithm
        - By merging version of an algorithm
        - Cover many concrete implementations
    - Make as abstract as possible
        - Maximum reusability
        - But efficient, concrete implementations
    - Define data abstractions: concepts
        - Constraints on parameters to templates. 

What is a concept:
    - Represent patterns in the requirements
    - Often different algorithms impose the same requirements on data structures
    - Describe logical abstractions in the domain
        - E.G. items can be assignable, containers
        - E.G. different types of iterators
    - A type can model the concept
        - ie. it satisfies the constraints
    - Generic programming gives a generic reusable implementation and a better understanding of the problem domain

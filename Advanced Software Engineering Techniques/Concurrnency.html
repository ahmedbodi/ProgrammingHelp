The world is concurrent:
    - Programs must handle simultaneous activities
        - Cant predict order of “requests”
        - Cant write a sequential program
    - Processor speed is limited
        - Parallelism is only way to increase speed

Handling concurrency:
    - Threads:
        - Simultaneously executing components
        - Use one thread to manage an external actor
    - Events:
        - Blocks of code triggered by an event
        - Keep event queue - handle events sequentially

Concurrent components:
    - Application
        - Complete program
        - OS may run several
    - Process
        - Code + Data + State of execution
        - Heavyweight
        - Has own memory space
    - Thread
        - Code + Data + State of execution
        - Lightweight
        - Shares memory space with there threads

Other terms:
    - Processes
        - Activity 
        - Task
    - Thread-Like co-operatively scheduled
        - Fiber
        - Co-routine


Parallel == Concurrency
    - Parallel systems
        - Multiple processors
    - Concurrent systems
        - Executing multiple programs at the same time, may see progress on several programs by sampling

All programs are to be treated as parallel


Concurrency is essential. Concurrent activities are treated as if they are parallel. Java Provides threads. Threads/Processes are concurrent program parts with threads sharing memory space and global variables. There can be consistency problems unless care is taken.

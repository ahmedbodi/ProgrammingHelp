What is a finite state machine:
    - A system with 
        - A limited number of internal states (modes)
        - These control its response to an input
            - In different states it may react differently to an input
    - Describe an FSM by defining each state
        - The response (action) and next state (state transition) generated by an input (event)
    - Representing an FSM:
        - By using a state transition diagram
        - By using a table

Why use a Finite State Machine:
    - Clear description of behaviour
    - Diagrams can be easier to understand than text
    - Simple and clear semantics (meaning) 
    - Allow checking for desirable properties
        - E.g. deadlocks (system trapped in a group of states)

Applications of FSM:
    - Digital Logic
    - Specifying data communication protocols
    - Specifying user dialogues
    - Defining class behaviour in UML
    - All computers are Finite State machines but because of large amounts of memory usually don’t treat as finite state machines to due to overwhelming complexity

FSM and User Interfaces
    - Used to define interaction:
        - States
        - Permitted user actions (FSM Events)
        - Responses (FSM Actions)
    - Identical input may give different outputs
        - E.G. closing word can be an empty file, modified file or saved file
        - Design FSM to represent these situations

FSM & Languages
    - FSM Can recognise sets of words
        - E.G. Identifiers, numbers , filenames
    - But not every possible set
        - Cant recognise matching brackets
    - Uses:
        -  Editor (global search and replace)
        - Pattern patching in JavaScript
        - Unix/Linux utility
            - GREP (find files containing a pattern
            - LE - generate a recogniser

Problems with FSM
    - Complexity
        - They can have many states and transitions
            - Allow variables and conditions
            - Modify the notation
                - Allow nested states
                - Allow actions that occur when a state is entered or left
    - Limited expressiveness
        - Cant define some behaviours
            - e.g. correct matching of brackets

FSM can model many systems:
    - Digital hardware
    - User interface
    - Communication protocols
    - Systems with modes
- Clear specification
- Allow reasoning
    - Checking for consistency and completeness
- Complexity and theoretical limitations

Grammer defines  language precisely:
    - Define programming languages
        - Help compiler writers
        - Help programmers
    - Verify program input
        - E.g. Check email address
    - Define data structure
        - XML Schemas define structure of XML file

Types of Grammar:
    - Context-free grammar
        - Set of possibly recursive definition of the form
    - Regular expression
        - Pattern made up of:
            - Sequences
            - Repetition
            - Alternatives
        - Languages recognised by a regular expression are the same as those that can be recognised by a finite state machine

Limitations of grammar types
    - Context free grammars
        - More complex to implement parsers
        - Messy for simple patterns
        - More powerful
            - e.g. can recognise properly matched brackets
    - Regular Expressions
        - Simple to implement
            - Included in scripting languages
        - Good for simple patterns
        - Cant recognise some patterns

A parse tree:
    - Depends on the input string
    - Depends on the grammar
    - The structure of the parse trees will control the meaning

A parser generator creates a program from a grammar and snippets of code

Regular expressions:
    - Are a way of defining simple text patterns
        - Ordinary characters (E.g. a - z)
        - Special characters (metacharacters)
    - Describes a pattern to look for in text
    - Applications
        - Editors e.g. more complex find and replace
        - Scripting language - input verification
        - Recognised keywords of a language
            - Front end to a context free parser
            - More efficient for simpler grammars
    - Can be represented as finite state machines

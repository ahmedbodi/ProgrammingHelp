Metrics

Updated:25/04/2017 11:39
Author:ahmedbodi@crypto-expert.com

Metrics is a measure of an interesting attribute
    - An attribute belongs to an entity
    - A metric attaches a value to an attribute
    - network - throughput - data transfer rate (MB/s)

Metrics are needed:
    - To compare features of different items
        - Is item x faster/more reliable/cheaper than Y
    - To assess the value of an individual item
        - How fast is item 2
        - Quality/Reliability/Cost
    - To make predictions
        - If X has a complexity cost of Y, how long will it take to develop
        - Is the design for module M too complex

What can be measured:
    - We can measure attributes of:
        - Products such a program’s efficiency/quality
        - Processes such as the methods used / cost effectiveness
        - Resources such as inputs to a process e.g. people - productivity or software tools - usability

Desirable features of a metric
    - Valid (actually measures the attribute)
    - Accuracy (measurement is close to true value)
    - Precision (small steps between measurements)
    - Reliable (Consistent and repeatable)
    - Can be manipulated (combine measures, take averages etc)

Metric scale type:
    - Nominal (classification: search, sort, calculation)
        - Value is a name - a category
    - Ordinal (first-1, second-2, third-3)
        - Difference between values doesn’t have a consistent meaning
    - Interval (e.g. a date)
        - Differences between values are meaningful
        - No fixed zero point
    - Ratio (e.g. execution time)
        - There is a fixed 0 point 
    - Absolute 
        - A count

Deciding what to measure:
    - Goal, Question, Metric
        - What is the purpose/goal of the investigation
        - What questions are relevant
        - What measures can answer the questions
            - Experimentally measured speed
            - Algorithmic Complexity
    - Other issues
        - Reliability, ease of measurement
        - Asking the right questions

What is complexity
    - Complexity has two meanings
    - Algorithmic complexity
        - Performance “efficiency”
        - Measures of time and space demands
        - Standard approach:asymptotic complexity

    - Module complexity
        - Measure of “difficulty” / understandability
        - No standard approach

Measuring Algorithm Efficiency
    - How long will the algorithm take
        - Why not just measure the program’s time
        - What factors affect Efficiency / Complexity
    - Desirable features of a complexity metric
        - Independant of programming language
        - Independent of hardware & implementation
        - Allows comparison of algorithms
        - Robust
        - Allows determination of time on any PC
            - Asymptotic complexity doesn’t do this

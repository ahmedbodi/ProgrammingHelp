Multi threading is similar to multi-processing

A multi-processing operating system can run several processes at the same time. Each process has its own address space and the OS scheduler decides when each process is executed. Separate processes do not have access to each others’s memory space

With Multi-Threading there are several points of execution within the same memory space. Each point of execution is called a thread and threads share access to memory. 

Why?
    - In a single threaded application one thread must do everything
        - If an application has several tasks to perform, those tasks will be performed when the head can get to them
            - A single task which requires a lot of processing can make the application feel unresponsive
    - In a multi-threaded application, each task can be performed on a operate thread so a long process does not slow down the application as other tasks are not waiting on it. 

Each thread is given its own context which consists of its own stack, virtual registers etc. The scheduler which is either virtual machine or OS based decides which thread executes at a given time. Each thread has a priority and the highest priority threads are priorities.

Parameter Passing:
    - Pass by value:
        - Example int add(int a, int b)
        - Make a local copy of a and b
    - Pass by pointer
        - Example int add(int *a, int *b)

    - Pass by reference
        - Example int add(int &a, int &b)

Passing classes as parameters:
    - If a class instance aka an object is passed by value the copy constructor will be used to make a copy. This is computationally expensive.
        - Example bool IsImageGreen(Image img)

        - By Pointer:
            - bool IsImageGreen (Image *img)
        - By Reference:
            - bool IsImageGreen (Image &img)

Dynamic Memory Allocation:
    - In C and C++, three types of memory are used by programs
    - Static memory
        - Where global and static variables live
    - Heap memory
        - Dynamicaly allocated at execution time, managed memory access using pointers
    - Stack memory
        - Used by stack variables

Types of program data:
    - Static Data
        - Allocated at compiler time
    - Dynamic data
        - Explicitly allocated and deallocated during program execution by C++ instructions using operators new and delete
    - Automatic data
        - automatically created at function entry, resides in activation frame of the function and is destroyed when returning from function. 

Dynamic Memory Allocation:
    - In C++, this is accomplished using new and delete operators. 
    - New is used to allocate memory during execution time
        - Returns a pointer to the address where the object is to be stored
        - Always returns a pointer to the type that follows new
        - If memory is available, in an area called the heap (or free store) new allocates the requested object or array and returns a pointer to the address of the memory allocated
        - Otherwise the program terminates with an error message
        - The dynamically allocated object exists until the delete operator destroys it
    - NULL Pointer
        - There is a pointer constant called the null pointer denoted by NULL
        - But NULL is not memory address is not 0
        - Trying to dereference a pointer will cause a crashed program
    - The Delete Operator    
        - The object or array currently pointed to by pointer is deallocated and the value of the pointer is undefined. The memory is returns to the free store.
        - It is a good idea to set the pointer to the released memory to NULL.
        - Square brackets are used after the delete operator to deallocate dynamically allocated array.

What is object orientated design:
    - Design is the identification of abstractions and mechanisms to produce desired behaviour
    - OO Design is the identification of objects or classes, their interactions and their inter-relationships to create a useful system

What is a good OO Design:
    - Cohesion
        - One well defined purpose or concept per class
    - Coupling
        - Minimal, controlled interaction between class
    - Re-use
    - Simplicity
    - Consistency

Basic OO Design Process:
    - Identify classes and objects at a given level of abstraction (what objects are in the specification)
    - Describe these classes and objects behaviour (what does each method do: methods and properties)
    - Describe the relationships between them (how do the objects interact)
    - Define the implementation of their behaviour in terms of other classes, objects and types. (Treat each object’s behaviour as a specification)
